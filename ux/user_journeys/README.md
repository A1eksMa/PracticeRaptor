# User Journeys: Пользовательские Сценарии

## 1. Введение

Данный раздел описывает основные сценарии взаимодействия пользователя с платформой PracticeRaptor. Каждый сценарий детализирует:

- **Контекст** — ситуация и мотивация пользователя
- **Задача** — что пользователь хочет достичь
- **Действия** — пошаговый путь к цели
- **Ожидания** — что пользователь ожидает от системы
- **Результат** — успешное завершение сценария

Все сценарии основаны на персоне **"Ученик" (Алексей)** — см. [user_persona.md](../user_persona.md).

## 4. Принципы описания сценариев

### 4.1. Структура каждого сценария

```
1. Контекст и мотивация
2. Задача пользователя (Job to be Done)
3. Предусловия
4. Пошаговые действия
5. Диаграмма (Mermaid)
6. Примеры интерфейсов (CLI, Telegram, Web)
7. Возможные проблемы и их решения
8. Критерии успеха
```

### 4.2. Фокус на пользователе

Каждый сценарий описывается с точки зрения Алексея:
- Что он думает и чувствует?
- Какие решения принимает?
- Что может пойти не так?
- Как система помогает достичь цели?
