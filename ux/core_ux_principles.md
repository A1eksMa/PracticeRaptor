# Core UX Principles

## 1. Цель

Данный документ определяет ключевые принципы, которые обеспечивают единообразие и консистентность пользовательского опыта (UX) на всех интерфейсах платформы PracticeRaptor (CLI, Telegram Bot, Web).

## 2. Ключевые Принципы

### 2.1. Функциональная Аналогия

*   **Описание:** Любое действие, которое пользователь может выполнить в одном интерфейсе, должно иметь прямое, понятное и легко обнаружимое соответствие в других интерфейсах. Это означает, что за аналогичными намерениями пользователя (например, «выбрать задачу со сложностью Easy») должны стоять вызовы одних и тех же базовых функций ядра.
*   **Реализация:**
    *   Определение унифицированного набора входных параметров для действий (например, фильтры для выбора задачи).
    *   Четкое сопоставление команд CLI, команд/кнопок Telegram и элементов UI в Web для каждого основного действия.
    *   Синхронизация представления результатов и доступных следующих действий.

### 2.2. Единообразная Обратная Связь

*   **Описание:** Пользователь должен получать четкую, консистентную и предсказуемую обратную связь о состоянии системы и результатах своих действий, независимо от используемого интерфейса.
*   **Реализация:**
    *   **Статус выполнения кода:** Результаты проверки кода (Accepted, Wrong Answer, Error, Timeout) должны быть представлены единообразно, с указанием критически важных деталей (например, входные данные, ожидаемый и фактический результат для `Wrong Answer`).
    *   **Системные сообщения:** Информация о загрузке, успешном завершении операции, ошибках (например, сетевых, валидации) должна иметь стандартный формат и стиль.
    *   **Визуальная/текстовая индикация:** Использование цветового кодирования (где применимо), понятных сообщений и индикаторов прогресса.

### 2.3. Унифицированный Доступ к Информации и Навигации

*   **Описание:** Структура приложения, информация и навигация по основным разделам должны быть логически связаны и интуитивно понятны на всех платформах. Пользователь, знакомый с навигацией в одном интерфейсе, должен легко ориентироваться в другом.
*   **Реализация:**
    *   Разработка единой Информационной Архитектуры (IA) и Sitemap.
    *   Обеспечение схожей последовательности действий для доступа к основным разделам (каталог задач, профиль, настройки, решение задачи).
    *   Использование понятных и последовательных терминов для обозначения разделов и функций.

### 2.4. Следование Ядру (Core Adherence)

*   **Описание:** Все действия пользователя, инициированные в любом интерфейсе, должны быть спроектированы таким образом, чтобы они в конечном итоге вызывали соответствующие функции и сервисы в ядре (Core) PracticeRaptor. Это гарантирует единую бизнес-логику и целостность данных.
*   **Реализация:**
    *   UX-дизайн должен быть тесно связан с API ядра.
    *   Любое взаимодействие с пользователем, которое влияет на состояние системы (например, отправка решения, изменение настроек), должно проходить через соответствующие порты и сервисы ядра.

## 3. Применение Принципов

При проектировании каждого конкретного интерфейса (CLI, Telegram, Web) эти принципы должны служить руководством. Фокус на **Stage 1.6 (CLI Enhancement)** будет заключаться в детальной реализации этих принципов для CLI, с созданием концептуальных аналогов для других интерфейсов, закладывая основу для их последующей полноценной разработки.
