# Функциональные требования к Telegram-боту LeetCode-аналогу

## 1. Общие положения
Проект представляет собой Telegram-бота, имитирующего функциональность платформы LeetCode, позволяя пользователям решать алгоритмические задачи.

## 2. Настройки и выбор задач

### 2.1. Настройки пользователя
Пользователь может настроить следующие параметры:
*   **Тематика задач:** Строки, Массивы, Связанные списки (выбор из предустановленного списка).
*   **Сложность задач:** Легкий, Средний, Сложный уровень.
*   **Язык программирования:** В прототипе - Python 3. Планируется расширение линейки языков в будущем.

### 2.2. Получение списка задач
*   После выбора настроек пользователь получает список из `N` задач (где `N` будет определено опытным путем, ориентировочно от 5 до 10), соответствующих выбранным критериям.
*   Список задач формируется один раз на основе примененных фильтров.
*   Пользователь может листать список задач вперед и назад с помощью кнопок навигации.

### 2.3. Навигация по списку задач
*   **Кнопки "Назад" и "Вперед":** Для пролистывания списка задач.
*   **Кнопка "Фильтр":** Возврат к шагу отбора задач по критериям.
*   **Команда `/list`:** Переход к списку доступных задач с учетом выбранных фильтров.
*   **Команда `/filter` или `/select`:** Переход к выбору фильтров и условий отбора задач.

### 2.4. Рандомная задача
*   В меню отбора по критериям доступна кнопка "Рандомная задача".
*   При нажатии на эту кнопку пользователь сразу переходит к этапу решения одной случайной задачи, соответствующей выставленным фильтрам, минуя шаг листинга.

## 3. Этап решения задачи

### 3.1. Представление задачи
*   Пользователь получает условие задачи.
*   Предоставляются 1-3 примера, демонстрирующие логику решения.
*   В конце условия задачи отображается сигнатура функции, которую необходимо реализовать.

### 3.2. Вспомогательные функции
*   **Команда `/help`:** Предоставляет теоретическую информацию или развернутую подсказку по алгоритму решения задачи.
*   **Команда `/solutions`:** Позволяет просмотреть список готовых решений (включая решения других пользователей, успешно прошедших задачу).

### 3.3. Ввод и проверка решения
*   Пользователь вводит свою реализацию функции в поле ввода бота и отправляет ее.
*   **Кнопка "Проверить":** Запускает быстрый тест решения на примерах, описанных в постановке задачи.
*   **Кнопка "Отправить":** Запускает полный набор тестов.

### 3.4. Обработка ошибок и результатов
*   **Проверка синтаксиса:** Если синтаксис неверный, возвращается трейсбэк ошибки и кнопка "Попробовать еще раз". При нажатии пользователь возвращается к предыдущему шагу.
*   **Тестирование решения:** При верном синтаксисе запускаются тесты, проверяющие обычные и граничные условия. Тесты останавливаются при первом непрошедшем тесте.
*   **Результаты тестов:** Каждый тест, возможно, будет снабжен пояснением (описанием теста, условий, ожидаемых и фактических результатов).
*   **Добавление тесткейсов:** В случае непрошедшего теста, вместе с описанием теста, должна быть возможность добавить его в пул проверки.
*   **Успешное прохождение:** При успешном прохождении всех тестов пользователь получает приветственное сообщение и статистику по времени и затраченной памяти.

### 3.5. Навигация во время решения
*   Во время выполнения задания и после успешного завершения у пользователя должна быть возможность вернуться к списку задач или выбрать другую/следующую задачу.

## 4. Будущие возможности (Этап технической зрелости)

*   **Статистика прохождений:** Сохранение и отображение статистики прохождений пользователя (какие задачи решены, какие нет).
*   **Дополнительные фильтры:** Для зарегистрированных пользователей будут доступны фильтры, например, "показывать только не решенные ранее задачи".
*   **Регистрация пользователей:** Возможность регистрации и авторизации пользователей.
*   **Просмотр статистики:** Просмотр общей статистики и прогресса пользователя.
