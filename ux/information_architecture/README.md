# Информационная Архитектура и Структура Навигации

## 1. Введение

Этот документ определяет информационную архитектуру (IA) и структуру навигации платформы PracticeRaptor. IA отвечает за организацию, структурирование и маркировку контента и функциональности приложения таким образом, чтобы сделать их понятными и легкодоступными для пользователей.

Наша цель — создать единую ментальную модель приложения, которая будет сохраняться при переходе между различными интерфейсами (CLI, Telegram Bot, Web), обеспечивая **аналогичность доступа к функциям и информации**.

## 2. Основные Принципы IA и Навигации

*   **Иерархичность и Структурированность:** Информация организована логично, с четкой иерархией разделов и функций.
*   **Консистентность:** Терминология, структура и пути доступа к основным разделам должны быть максимально схожими во всех интерфейсах.
*   **Обнаруживаемость:** Все ключевые функции и информация должны быть легко находимы.
*   **Предсказуемость:** Пользователь должен понимать, где находится и как перейти к другим разделам.

## 3. Обзор Структуры Приложения

Детальная структура разделов и функций приложения описана в `sitemap.md`.

## 4. Навигация по Интерфейсам

В данном разделе описываются общие принципы навигации для каждого интерфейса:

*   **CLI:** Командная строка, иерархия команд, использование аргументов и опций для навигации и фильтрации.
*   **Telegram Bot:** Команды бота, inline-кнопки, управление состоянием диалога (FSM).
*   **Web:** Меню навигации, хлебные крошки, формы фильтрации, ссылки.

Конкретная структура команд, кнопок и элементов UI определяется в документации соответствующих интерфейсов (`ux/interfaces/`).
